# Acceptance criteria

- A complete routine case is auto-cleared and audited.
- An after-hours restricted-area case enters escalated human review.
- A defined prompt-injection case is blocked before review-model execution.
- An incomplete request is returned for correction without review-model execution.
- A reviewer can approve, reject, or request more information on eligible cases.
- Every case exposes a chronological audit timeline.
- The MCP server exposes only documented read-only tools.
- The 30-case deterministic evaluation script completes and produces JSON and Markdown reports.
- No real personal data or proprietary AUREX SICOS logic is present.
